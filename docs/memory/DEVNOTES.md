# TinyRTC Development Notes

This directory contains project-specific development notes and memory.

## Structure:

- `YYYY-MM-DD.md - Daily development logs (what was done, decisions made)
- `ARCHITECTURE_NOTES.md - Architecture decisions and tradeoffs
- `API_CHANGES.md - API change history
- `BUGS.md - Known issues and bug tracking

## Purpose

Keeping development notes within the project allows independent iteration,
avoiding context pollution from other projects in the global workspace.

All development discussions and decisions are documented here.
